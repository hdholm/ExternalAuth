El plugin de Autenticación Externa soporta el inicio de sesión basado en la identidad proporcionada por el servidor web (alias el usuario remoto), que normalmente se suministra a través de servidores proxy o módulos de servidor web (por ejemplo. "Basic Auth"). Difiere de la opción de Piwigo "apache_authentication" al proporcionar un mayor grado de flexibilidad, incluyendo:

  • Control sobre qué variable(s) se utilizan para la autenticación
  • Piwigo login siempre seguirá cambios en (reconocido) Usuarios remotos
  • Nativo Piwigo logins posible cuando el usuario remoto se considera un invitado
  • Opción de copiar las contraseñas suministrados servidor web a las cuentas de Piwigo
  • Urls de inicio de sesión externos que reemplazan o coexisten con Urls nativos
  • Posibilidad de auto-registrar Usuarios Remotos desconocidos como nuevos usuarios de Piwigo
  • Control sobre perfiles de auto-registro, contraseñas, estado y notificaciones
  • Lista flexible de Usuarios Remotos considerados huéspedes

La Autenticación Alternativa es opcional, y permite el ingreso nativo de Piwigo cuando el Usuario Remoto actual es considerado un invitado. NOTA: Si el registro automático del usuario remoto está desactivado, cualquier usuario remoto desconocido para Piwigo se considera un invitado. Si Fallback está desactivado, los accesos a Piwigo siempre coincidirán con el usuario remoto actual.

El plugin hace todo lo posible para evitar el bloqueo de cuentas no intencional, y siempre se desactiva al activarse para permitir la configuración antes de que la aplicación de inicio de sesión esté activada (¡lo que puede desconectar inmediatamente la sesión actual!).
