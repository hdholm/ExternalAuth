Das Plugin Externe Authentifizierung unterstützt Login basierend auf der Identität des Webservers (auch Remote User genannt), der normalerweise über Proxy-Server oder Webserver-Module (z.B. "Basic Auth") bereitgestellt wird. Er unterscheidet sich von der Option Piwigo "apache_authentication" dadurch, dass er ein höheres Maß an Flexibilität bietet, unter anderem:

  • Kontrolle darüber, welche Variable(n) für die Authentifizierung verwendet werden
  • Piwigo login wird immer Änderungen folgen in (erkannten) Remote User
  • Native Piwigo Logins sind möglich, wenn der Remote User als Gast angesehen wird
  • Option zum Kopieren von Webserver lieferte Passwörter an Piwigo-Konten
  • Externe Login/Logout-Urls ersetzen oder koexistieren mit nativen Urls
  • Möglichkeit, um unbekannte Remote-Benutzer automatisch als neue Piwigo-Benutzer zu registrieren.
  • Kontrolle über Auto-Registrierungs-Profile, Passwörter, Status und Benachrichtigungen
  • Flexible Liste Remote-Benutzer als Gäste anzuerkennen

Fallback Authentication ist optional und erlaubt native Piwigo Logins, wenn der aktuelle Remote User als Gast gilt. HINWEIS: Wenn die automatische Registrierung eines entfernten Benutzers deaktiviert ist, gilt jeder entfernte Benutzer, der Piwigo unbekannt ist, als Gast. Wenn Fallback deaktiviert ist, werden Piwigo Logins immer mit dem aktuellen Remote User übereinstimmen.

Das Plugin unternimmt jede Anstrengung, um eine nicht absichtliche Sperrung des Accounts zu verhindern, und ist immer deaktiviert, um eine Konfiguration zu ermöglichen, bevor die Login-Aktivierung aktiviert ist (was sofort die aktuelle Session ausloggen kann!).
